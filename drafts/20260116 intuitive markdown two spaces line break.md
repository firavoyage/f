Yes, Fira, some people do that—but it’s mostly specific to **Markdown**, the language used in GitHub README files, comments, and wikis.

In Markdown, if you want a line break _without starting a new paragraph_, the simple rule is:

- End a line with **two spaces**, then hit **Enter**.

Example:

```
This is the first line.␣␣
This is the second line.
```

Renders as:
This is the first line.
This is the second line.

Without the two spaces, Markdown usually merges the lines into a single paragraph.

It can feel odd at first—especially if you’re used to just pressing Enter—but it’s a lightweight way to control line breaks without using extra HTML tags like `<br>`.

Some people prefer using HTML `<br>` instead because it’s more visible in the code, but the double-space trick is quite common on GitHub.
