If you do not start a new line, appending after text means writing to its right.

## what about append to existing text

"Append to existing text" also implies writing to the right, but it is **much more precise** than "append after."

In both everyday English and programming, "append to" means you are attaching something directly to the back of the original object on the same line.

### Why "Append To" Means "To the Right"

- **Direction:** Since English moves left-to-right, attaching text _to_ a word automatically forces the new text to the right side.
- **No Gaps:** It implies the new text becomes part of the original line, rather than starting a new line below.

### Examples

- **Text:** `Apple`
- **Action:** Append `sauce` to the existing text.
- **Result:** `Applesauce` (the new text is on the right).

### When it might NOT mean to the right

The only exception is if the "existing text" is a multi-line document (like an essay or a log file). In that case, "append to the file" means adding text to the very bottom.

- **How it works**: The descriptions form a single, unbroken vertical wall down the entire terminal screen.
- **Pros**: Incredibly satisfying and neat geometric layout.
- **Cons**: If one usage string is 40 characters long, the short options (`-h`, `-v`) will have a massive, awkward blank space before their descriptions start.

***

### Industry Standards: What the Tools Do

Almost all major CLI frameworks (like Go's **Cobra**, Rust's **Clap**, and Python's **Click**) choose **Option 1 (Section-Specific Alignment)**. They dynamically calculate the longest element in each individual block to save horizontal screen real estate.

### Implementation Checklist

1. Find the longest string in the left column for **just that section**.
2. Add **2 to 4 spaces** of padding to that length. This is your target description column for that section.
3. If any line exceeds a reasonable threshold (like 30 or 40 characters), wrap the description to the _next_ line instead, indented to match the description column.

# . link 0.0

```sh
link(){
  local reset=$(tput sgr0)
  local bold=$(tput bold)
  local italic=$(tput sitm)
  local underline=$(tput smul)
  local heading=""
  # local heading=$italic
  # local heading=$underline

  local version="link 0.0 (2026.07.12)"
  local help=$(cat <<- EOF | sed 's/^  //'
  Create and check symlinks

  ${bold}${heading}Usage:${reset}
    ${bold}link${reset} <source> <target>    Create symlink
    ${bold}link${reset} <target>             Check symlink
    ${bold}link${reset} [flag]               Check version or help

  ${bold}${heading}Options:${reset}
    ${bold}-v, --version${reset}             Print version
    ${bold}-h, --help${reset}                Print help
	EOF
	)

  if test $# -eq 1; then
    if test $1 = "--help" -o $1 = "-h"; then
      echo $help
    elif test $1 = "--version" -o $1 = "-v"; then
      echo $version
    else
      ls -l $1
    fi
  elif test $# -eq 2; then
      ln -snf $1 $2
  else
    echo $help
  fi
}
```
