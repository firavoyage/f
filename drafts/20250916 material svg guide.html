SVG Guide

Learn how to create and manipulate Scalable Vector Graphics

What is SVG?

SVG (Scalable Vector Graphics) is an XML-based markup language for describing two-dimensional vector graphics. Unlike raster images (like JPG or PNG), SVG graphics don't lose quality when zoomed or resized.

<svg width="200" height="200" viewBox="0 0 200 200"> <circle cx="100" cy="100" r="80" fill="#4cc9f0" /> <text x="100" y="110" text-anchor="middle" fill="#0d1b2a" font-size="24">SVG</text> </svg>

Note: SVG is supported in all modern browsers and can be created with any text editor, but specialized tools like Adobe Illustrator or Inkscape can also be used.

Advantages of SVG

  • Scalability without quality loss
  • Small file size for simple graphics
  • Styleable with CSS
  • Animatable with CSS or JavaScript
  • Accessible (text content is available to screen readers)
  • Search engine friendly
  • Can be created and edited with any text editor

Common Use Cases

  • Icons and logos
  • Data visualizations (charts, graphs)
  • Interactive maps
  • Animations and illustrations
  • Complex UI elements

Basic SVG Shapes

SVG provides several basic shape elements:

Rectangle

<rect width="80" height="60" x="10" y="20" fill="#4cc9f0" />

Circle

<circle cx="50" cy="50" r="40" fill="#f72585" />

Ellipse

<ellipse cx="50" cy="50" rx="40" ry="30" fill="#7209b7" />

Line

<line x1="10" y1="10" x2="90" y2="90" stroke="#4cc9f0" stroke-width="4" />

SVG Paths

The <path> element is the most powerful and complex SVG shape. It can be used to create any shape by combining lines, curves, and arcs.

<path d="M 10 100 Q 100 10 190 100 T 370 100" stroke="#4cc9f0" stroke-width="4" fill="none" /> <path d="M 10 150 C 50 50, 150 50, 190 150" stroke="#f72585" stroke-width="4" fill="none" />

Path Commands

  • M - Move to
  • L - Line to
  • H - Horizontal line
  • V - Vertical line
  • C - Cubic Bezier curve
  • Q - Quadratic Bezier curve
  • A - Arc
  • Z - Close path

Styling SVG

SVG elements can be styled using attributes or CSS. Common styling properties include:

Fill

fill: #4cc9f0;

Stroke

stroke: #f72585; stroke-width: 8;

Opacity

opacity: 0.7;

Gradient

fill: url(#grad1);

SVG Animation

SVG can be animated using CSS, JavaScript, or SMIL (Synchronized Multimedia Integration Language).

<circle cx="100" cy="100" r="30" fill="#4cc9f0"> <animate attributeName="r" from="10" to="40" dur="2s" repeatCount="indefinite" /> </circle> <rect x="70" y="70" width="60" height="60" fill="#f72585" opacity="0.7"> <animateTransform attributeName="transform" type="rotate" from="0 100 100" to="360 100 100" dur="3s" repeatCount="indefinite" /> </rect>

Animation Approaches

  • CSS Animations: Use @keyframes and transition properties
  • JavaScript: Use requestAnimationFrame or libraries like GSAP
  • SMIL: Native SVG animation elements

SVG Best Practices

  • Optimize SVG files by removing unnecessary metadata
  • Use relative units for better scalability
  • Minify SVG code for production use
  • Provide meaningful IDs and classes for styling
  • Consider accessibility with proper titles and descriptions
  • Test across browsers for compatibility

Tools for Working with SVG

  • Design Tools: Adobe Illustrator, Inkscape, Figma
  • Optimization: SVGOMG, SVGO
  • Libraries: D3.js, Snap.svg, SVG.js
  • Animation: GSAP, Anime.js

Pro Tip: You can inline SVG directly in HTML for better performance or reference external SVG files with <img>, <object>, or CSS background-image.
