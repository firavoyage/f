Here is how to decide between the two:

### Use "Match" for Specific Examples

Use **match** when you want the new code to look exactly like another existing file, function, or component.

- "Refactor the component to **match** the existing button implementation."
- "Update the API response structure to **match** the login endpoint."

### Use "Follow" for Rules and Guidelines

Use **follow** when you want the code to obey written instructions, design systems, or team conventions.

- "Please rewrite this function to **follow** our established coding standards."
- "Update the styling to **follow** the design system guidelines." \[[1](https://www.linkedin.com/pulse/design-systems-vs-style-guides-understanding-impact-reyhan-2n4qc)]

- **Match** = Look exactly like this other piece of code.
